# 🚀 阿里云宝塔面板部署与多端数据实时互通极速配置指南

本指南将教你如何**从零开始**在阿里云宝塔面板（BT Panel）中一步一步重新部署系统，并配置 `admin.lyheiwandaijiamax.com`（管理后台）与 `lyheiwandaijiamax.com`（手机端/乘客扫码端），实现所有设备（预览端、APP端、网页端）数据 100% 实时互通互联。

---

## 🗺️ 多端架构与域名划分

为了保证系统最高效的运行，我们将业务做如下分工：

1. **管理后台 + 数据中心 (API)** ➜ 使用域名：`admin.lyheiwandaijiamax.com`
   * 这是你的 **Node.js 核心后端应用程序**。
   * 负责存储所有数据（司机、乘客、订单、系统设置），处理短信获取、品牌配置等所有 API 请求。
   * 预览端调试、手机APP、模拟器都会自动和这个端点交互，实现完全的数据物理同源。
   
2. **乘客自助端 (代开单静态页)** ➜ 使用域名：`lyheiwandaijiamax.com`
   * 负责司机二维码扫码后，乘客打开的自助下单网页（即 `passenger_order.html`）。
   * 它将无缝跨域连接到 `admin.lyheiwandaijiamax.com` 获取数据，确保手机扫码与后台数据完全秒级同步。

## 🛠️ 第一部分：宝塔面板环境准备（仅需3步）

如果你已经清空了宝塔面板，请先登录宝塔后台，完成以下基本软件的安装：

### 1. 安装 Nginx
* 进入宝塔 **【软件商店】** ➜ 搜索 **Nginx** ➜ 安装任意版本（推荐安装 Nginx 1.22 或 1.24 极速安装）。

### 2. 安装 Node.js 版本管理器
* 在 **【软件商店】** 中搜索 **Node.js** ➜ 安装 **Node.js项目管理器**（或 PM2 管理器）。
* 打开 Node.js 项目管理器，选择安装 **Node.js 官方 LTS 版本（推荐 v18 或 v20）**。

### 3. 创建 MySQL 数据库（如果你要使用自建数据库）
* 点击宝塔左侧 **【数据库】** ➜ **【添加数据库】**：
  * 数据库名：`daijia` (可自选)
  * 用户名：`daijia`
  * 密码：`你的安全密码`
* **【重要说明】**：系统启动后，后台会自动检测数据库配置：
  * 如果你在 `.env` 文件里填写了你的本地 MySQL 数据库，系统会自动切换到本地并自动创建数据表（包含自动从云端同步历史数据），全过程零手动操作。
  * 如果没有配置本地 MySQL，系统也会默认使用内置云端数据源安全通信，双重兜底。

## 📦 第二部分：解压与配置文件

### 1. 下载 `daijia_deploy.zip`
你可以通过以下任一方式获取最新编译生成的打包文件：
* **方式 A (推荐)**：在 **AI Studio 控制台右上方** ➜ 点击 **【设置 (Settings)】** ➜ 选择 **【Export to ZIP】** 导出整个项目。这样可以获取最完整、最安全的打包源码。
* **方式 B (直接浏览器下载)**：在浏览器中直接打开你的开发/预览地址：
  `https://ais-dev-nstmvaox4sb7kditx7suvv-11329907111.asia-east1.run.app/daijia_deploy.zip`
  *提示：此文件是由我们用最优 ZIP_DEFLATED 压缩算法打包的，已经内置了编译好的 `dist/server.cjs` 后端程序。*

### 2. 上传并解压
* 登录宝塔 ➜ 点击左侧 **【文件】** ➜ 进入网站主目录（如 `/www/wwwroot/daijia_server`，如果目录不存在可以新建）。
* 将 `daijia_deploy.zip` 上传到该目录并解压。

### 3. 配置 `.env` 环境文件
在解压出的根目录下找到 `.env` 文件（如果没有，可以复制 `.env.example` 并重命名为 `.env`）：
* 双击 `.env` 并在其中填入你阿里云的**短信密钥**和刚才在宝塔里新建的 **MySQL 数据库配置**：

```env
# 1. 阿里云短信服务的真实 AccessKey (用于给手机下发真实短信)
ALIBABA_CLOUD_ACCESS_KEY_ID=你的真实阿里云_ACCESS_KEY_ID
ALIBABA_CLOUD_ACCESS_KEY_SECRET=你的真实阿里云_ACCESS_KEY_SECRET

# 2. 本地自建 MySQL 数据库配置（填入后，系统会自动切换成 MySQL 并自动建表，100%全自动）
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=daijia
MYSQL_PASSWORD=你的数据库密码
MYSQL_DATABASE=daijia
```

## 🚦 第三部分：启动 Node.js 后端服务

我们使用宝塔的 **Node.js项目管理器** 或者是 **PM2 命令行** 运行程序：

### 选项 A：使用宝塔【Node.js 项目管理器】图形界面（推荐）
1. 打开宝塔 **【Node.js项目管理器】** ➜ 点击 **【添加项目】**。
2. **项目路径**：选择你刚才解压的目录（如 `/www/wwwroot/daijia_server`）。
3. **启动文件**：选择 `dist/server.cjs`（注意是 dist 目录下的 server.cjs 文件）。
4. **项目名称**：输入 `daijia-backend`。
5. **端口**：填写 `3000`。
6. **运行用户**：默认即可（如 www）。
7. **绑定域名**：先留空（我们后续用网站反向代理更安全稳定）。
8. 点击 **【提交】** 启动项目。如果状态显示为“运行中”，说明后端已经成功在阿里云 3000 端口跑起来了！

### 选项 B：使用 PM2 命令行（适合喜欢终端操作的用户）
在你的解压目录下打开宝塔内置终端，输入以下命令：
```bash
# 1. 安装生产依赖
npm install --production

# 2. 启动服务（后台24小时不挂断运行）
pm2 start dist/server.cjs --name "daijia-backend"

# 3. 查看启动状态
pm2 status
```

## 🌐 第四部分：配置双域名与反向代理（实现数据互通）

这是最核心的一步，实现两个域名各司其职，而数据实时共享：

### 步骤一：配置 `admin.lyheiwandaijiamax.com` (后台/API 域名)
1. 在宝塔面板左侧点击 **【网站】** ➜ **【添加站点】**。
2. 填写域名：`admin.lyheiwandaijiamax.com`。
3. 根目录：可以直接指定到刚才解压后端目录的 `dist` 文件夹（如 `/www/wwwroot/daijia_server/dist`）。
4. 点击提交创建网站。
5. **配置反向代理（极其重要）**：
   * 点击该网站右侧的 **【设置】** ➜ 找到左侧 **【反向代理】** ➜ 点击 **【添加反向代理】**。
   * 代理名称：填 `daijia_api`
   * 目标 URL：填写 `http://127.0.0.1:3000` (把外部请求导向我们刚才跑在 3000 端口的后端程序)
   * 点击 **【保存】**。
6. **配置 SSL 证书**：
   * 点击该网站设置 ➜ 选择左侧 **【SSL】** ➜ 勾选你的域名 ➜ 点击一键申请免费证书，开启 **强制 HTTPS**。

---

### 步骤二：配置 `lyheiwandaijiamax.com` (乘客下单端/司机二维码域名)
1. 点击宝塔 **【添加站点】**。
2. 填写域名：`lyheiwandaijiamax.com`（你可以同时把 `www.lyheiwandaijiamax.com` 也写上）。
3. 根目录：直接指定到你刚才上传解压的那个目录（因为下单网页 `passenger_order.html` 就直接在这个目录下）。
4. **配置默认文档（极其重要）**：
   * 乘客扫码后应该默认加载 `passenger_order.html` 页面。
   * 点击该网站右侧的 **【设置】** ➜ 找到 **【默认文档】**。
   * 在最上方添加一行：`passenger_order.html`，然后点击添加（确保它在第一位优先加载）。
5. **配置 SSL 证书**：
   * 点击该网站设置 ➜ 选择左侧 **【SSL】** ➜ 勾选你的域名 ➜ 一键申请免费 SSL 并开启 **强制 HTTPS**。
